Speedy Archiver Component

created by TEAM Informatics, Vedran Stanic December 2010

Summary

This component allows a "sysadmin" user to download an archive as a zip file and upload it to another instance by adding a new link to the main OOTB menu called Speedy Archiver essentially speeding up the daily changes deployment process.

Detailed Description


By using a similar concept as the OOTB CMU bundles, now you can use solely the front end OOTB pages (sysadmin is the only one that sees the extra menu) to deploy changes from one CS instance to another. For example let's say that you have 6 environments where the code needs to be pushed to: TIDEV, DEV, TEST, TRAIN, UAT, PROD. What you would do with this component is the following:

1. Using Archiver create an archive called A1 on all environments
2. Using Archiver define the details of the archive on TIDEV and export it
3. Navigate to the TIDEV "Speedy Archiver" home page and download A1 archive by clicking on it's link
4. Navigate to the X environment's "Speedy Archiver" home page and upload the downloaded "A1.zip" file while also selecting A1 from the dropdown box
5. Using the X's Archiver import the A1 archive.

In the background what is really happening is that we are compiling a zip file of the archive job and downloading it, then we are uploading it and overriding the existing archive folder with the contents of the zip file all this using Java.

Requirements and Assumptions

The following are required:

  1. UCM 10gR3 or higher.
  2. Any browser

Warnings and Potential Conflicts with Core Resources

1. Do not use important archive jobs - rather make "simple temporary ones" because the component deletes the archive folder during the upload. This component is not tested for full stability so therefore be cuatious. 2. Make sure you select the correct archive name from the dropdown while uploading as that archive folder will be replaced. 3. This component overrides the following ootb include: custom_finish_layout_init
